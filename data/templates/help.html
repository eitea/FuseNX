  • Guide

    Installing restic
    Go to restic download page and download restic. Move it somewhere and add the directory to the Path environment variable.
    Where to start?
    First go to the Repository section. Create a new Repository. Make sure you have restic installed.
    Creating a Repository
    If you are using a local backup, type an absolute path like "C:\backup" or an relative path e.g. "backup". A relative path creates a repository relative to this program. Be sure to remember your restic password.
    If you are using other types of repositories, look at the tips at the bottom and here.
    What next?
    Head on over to Backup Jobs and create a new one. Be sure to include at least one folder to start with. Then you can save and browse more files to include in your Backup.
    If this is your first time, try repeating every minute and see if your backup turns green. If not, you might have started FuseNX without administrator rights, put in a too small repeat (less than 1 minute) or set the start date too far in the future or past.
    Settings and Email
    Go to Settings to customize FuseNX. It is recommended to set up email so you will be notified when backups complete. TLS is used when sending mails. Here are a few examples.
    Gmail

    Hostname: smtp.gmail.com
    Port: 587
    Username, From, Recipient: your email address
    Password: your password

    Outlook

    Hostname: smtp-mail.outlook.com/smtp.live.com
    Port: 587
    Username, From, Recipient: your email address
    Password: your password

  • Issues

    My Snapshots won't show up immediately
    Creating snapshots is handled in the Background. If your Backup Job contains many large files, wait a few minutes before refreshing the page.
    My Backup Jobs aren't working
    If scheduled tasks won't run, add the restic executable to PATH. If this still doesn't work, move the restic executable next to this executable.
    Also FuseNX needs Administrator rights to interact with scheduled tasks.
    I forgot my password
    Losing your password means that your data is irrecoverably lost. If you have set up SMTP, click on "Forgot Password". This only affects FuseNX. Your data in restic repositories is safe if you still have the restic password.
